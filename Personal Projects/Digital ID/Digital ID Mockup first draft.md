Digital IDs have been in news lately, as the government attempts to implement them. However, there have been considerable backlash against this scheme, owing to privacy and security concerns. The public generally do not trust the government to keep a large database of IDs secure, and do not trust that such a system would not be used to track and spy on citizens.

Sharing some of these concerns, I decided to investigate if a digital ID scheme that addresses these concerns while still functioning as a method of identity verification would be theoretically possible.

For such a system to satisfy the public while remaining secure and robust certain principles need to be upheld:
- Only the government can issue valid IDs
- Verification of IDs is possible by anyone and does not require any contact with the government
- When required to verify certain information using the ID, only the minimum required information is shared, rather than the full ID
- The entire system is transparent and open-source

This is possible with an Attribute-Based Credential system which allows for Selective Disclosure and Zero-Knowledge proofs.
This is made possible by using BBS signatures, named after Dan Boneh, Xavier Boyen, and Hovav Shacham. In this scheme, a Signer (the government) signs multiple messages corresponding to different fields in the ID (name, gender, etc) and produces a fixed size signature. A Prover (the holder of the ID) can generate a proof whereby they choose which messages to disclose, while revealing no information about the other messages. The proof guarantees the integrity and authenticity of the disclosed message (e.g. that they were signed by the government).
BBS signatures also allow for unlinkable proofs, meaning that the proof that was generated by the Prover cannot be traced back to the original message, even by the Signer (the government).

This scheme uses public-key cryptography, where the government acting as the Signer keeps a secret private-key, which is used to generate a public-key which is published. When an ID will issued, it is signed by the private-key; when a proof generated by the ID will be verified, it can be done so by using the published public-key. As long as the private-key remains secret, this scheme is robust.

# Real Life Implementation
In reality, the wallet would likely be an app or a feature on phones, a browser addon or an operating system feature on computers. Receiving the ID would likely include a secure QR code or other delivery system rather than copy-pasting. 

# Limitations

As it stands, this demo has certain limitations. More advanced systems which solve some of these issues are possible however. For example, instead of storing age in a credential, the date of birth can be stored. And when generating proofs, certain calculations such as `date_today - date_of_birth > 18 years` can be performed cryptographically, and only the binary fact that the user if over 18 is sent, rather than the age.
Another issue would be credential sharing. One user could send the credential package to another, allowing them to impersonate the other person. A solution to this may be a system where biometric data is linked to the credential when issuing it, and matching biometric data such as a fingerprint is required cryptographically when creating a proof. This system would however raise privacy concerns about government collection of biometric information.
Another issue is revocation of IDs. In this current system it is impossible to revoke a credential for any reason, which is something that would be needed in a real application.

# Conclusion
A privacy preserving digital ID is technically feasable, and while it would face further challenges which the simple demo presented here does not address, a more advanced system would be able to solve these. At any rate, such a digital ID would be much better for privacy and security of users than the current system of uploading physical ID card photos and selfies to unaccountable 3rd party companies, who upon being hacked, bought out, or facing financial difficulty may sell or leak millions of credentials. A digital ID scheme such as this one would also render tracking of user data impossible by both governments and corporations, while also being more robust than physical IDs. Creating fake digital IDs would be impossible, unlike fake physical IDs, which are frequent.
